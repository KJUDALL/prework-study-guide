Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
    • It is important to utilize semantic HTML.
  • The header values should always remain in order (ie. h1 through h6). This is especially helpful for accessibiliy purposes.
  • All of the information listed in the head element is NOT shown to the user.
  • HTML is like the foundational pieces of a house. Later, CSS will be used to paint the house and make it look visibly appealing. And lastly, JS will help make it completely functional, like having doors and windows that open, usable water and electricity, etc.

CSS

  • CSS is helpful in adding designs and graphics to a web application.
  • This language helps to add color and organization into a webpage
  • There are a lot of style options to choose from and CSS helps us dictate that and present a web page in very specific ways, depending on our style guide.
  • External Style Sheets are most common when using the CSS language. It allows developers to keep their CSS rules in a separate file, making it easier to change their page styling. You have to link this in the head section of the HTML.
  • When you add padding to an element, you are actually increasing the size of the whole element. Whereas, a margin simply adds space around said element. They can't be seen, but you will notice a difference to your elements when adding either of these.

Git

  • Git is like GitBash, but for MacOS. GitBash is for Windows.
  • Git and Git bash are one in the same; synonomous. Git is software on the computer that already exists, which tracks changes I make to code.
  • You can utilize Git commands and functions on a Mac by using the Terminal.
  • This allows you to communicate with the computer like a regular user would via a GUI. It also syncs up nicely to an online development software like GitHub.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull: fetches and merges code into the current branch you're working on.

ProTips

  • “The words “element” and “tag” are commonly used interchangeably in the web development world, but it is important to know the difference. An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.”
